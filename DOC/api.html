MediCare API 文档

Base URL: http://localhost:8080  |  认证方式: Session Cookie  |  版本: v1.0

认证流程

登录 → 鉴权 → 角色校验

POST /api/auth/login → Session 写入用户 → 后续请求携带 Cookie → AuthInterceptor 校验 → @RequireRole 角色校验

前端需设置 withCredentials: true 以携带 Cookie；
未登录返回 401，权限不足返回 400（BusinessException）。

挂号就诊流程

排班 → 挂号 → 叫号 → 写病历 → 开处方 → 完成就诊

创建排班 → POST /registrations → PUT /{id}/call → POST /medical-records → POST /prescriptions → PUT /{id}/complete

处方发药流程

开处方(自动扣库存) → 取药 或 作废(回滚库存)

POST /prescriptions → PUT /{id}/dispense 或 PUT /{id}/cancel

库存管理流程

入库(增加库存+日志) / 出库(安全扣减+日志)

POST /{id}/stock-in | POST /{id}/stock-out | GET /low-stock

认证 Auth

POST /api/auth/login 登录

请求体

字段类型必填说明
usernameString必填用户名
passwordString必填密码（明文，兼容 BCrypt）

响应

{ "code": 200, "message": "success", "data": { "id": 1, "username": "admin", "realName": "管理员", "role": "admin", "status": 1, "doctorId": null, "password": null } }
POST /api/auth/logout 登出

说明

销毁 Session，无需请求体。

GET /api/auth/current 获取当前登录用户

说明

前端刷新页面后调用此接口恢复登录态，未登录时 data 为 null。

用户 Users

GET /api/users 用户列表 admin

返回所有用户，密码字段置空。

POST /api/users 创建用户 admin

请求体 — SysUser

字段类型必填说明
usernameString必填用户名（唯一）
passwordString必填密码（BCrypt 加密存储）
realNameString可选真实姓名
roleString必填角色: admin / doctor / pharmacist
statusInteger可选状态(1=启用)
doctorIdLong可选关联医生ID
PUT /api/users/{id} 更新用户 admin
DELETE /api/users/{id} 删除用户 admin

超级管理员 admin 账号禁止删除。

PUT /api/users/{id}/password 修改密码

请求体 — ChangePasswordRequest

字段类型必填说明
oldPasswordString必填旧密码
newPasswordString必填新密码

只能修改自己的密码，验证旧密码正确后才允许修改。

统一响应格式

Result<T>

{ "code": 200, "message": "success", "data": { ... } }
字段类型说明
codeint状态码：200=成功，400=业务错误，401=未登录，403=禁止，409=冲突，500=服务器错误
messageString提示信息
dataT业务数据，失败时为 null

错误码说明

HTTP 状态码code说明
200200请求成功
400400业务异常 / 参数校验失败 / 请求体格式错误
401401未登录（AuthInterceptor 拦截）
403403禁止操作（如修改他人密码）
409409数据约束冲突 / 乐观锁冲突
500500服务器内部错误（信息脱敏）
