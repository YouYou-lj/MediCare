# MediCare 智慧医疗门诊管理系统 — 教师指导教案

> **课程名称**：JavaFX 桌面应用开发与分层架构实战  
> **项目案例**：MediCare 智慧医疗门诊管理系统  
> **课时**：12 课时（每课时 45 分钟）+ 课后分组开发时间  
> **授课对象**：有 Java SE 基础、希望掌握桌面应用与企业级架构的学员  
> **授课方式**：理论讲解 + 代码演示 + 学员实操（3-5 人小组协作）  
> **版本**：v2.0（脚手架 + 分组协作版）

## 一、教案核心设计

### 1.1 教学模式

本教案采用**"脚手架 + 分组协作"**的教学模式：

- **老师**：提供脚手架代码（基础框架），演示一个完整模块的开发过程，指导学生分组实现其余模块
- **学生**：以 3-5 人小组为单位，在脚手架上独立完成分配模块的 DAO → Service → Controller → FXML 开发
- **协作**：各小组通过 Git 分支协作，按模块依赖顺序集成联调

### 1.2 脚手架说明

老师向学生分发 `scaffold/` 目录，包含已封装好的基础框架代码。学生**禁止修改**脚手架中的文件。

**脚手架包含**：

| 类别 | 文件 | 说明 | 是否可修改 |
|------|------|------|-----------|
| 项目配置 | `pom.xml` | Maven 依赖、shade 插件 | ❌ |
| 入口类 | `Main.java`, `Launcher.java` | 应用启动 | ❌ |
| 工具类 | `ConnectionConfig.java` | HikariCP 连接池 | ❌ |
| 数据访问基类 | `BaseDAO.java` | 泛型 DAO + 事务辅助 | ❌ |
| 类型处理器 | `JavaTimeBeanProcessor.java` | LocalDate/LocalDateTime 映射 | ❌ |
| 实体模型 | `model/*.java` (11个) | 所有 POJO | ❌ |
| 主界面框架 | `MainView.fxml`, `MainController.java` | 左侧导航 + 内容区 | ⚠️ 仅添加注释 |
| 样式文件 | `css/*.css` | 全局样式 | ⚠️ 可扩展 |
| 日志配置 | `logback.xml` | SLF4J/Logback | ❌ |
| 数据库脚本 | `schema_baseline.sql` | 11 张表 DDL + 初始数据 | ❌ |

**学生需要创建**：DAO 层、Service 层、Controller 层、FXML 视图文件。

### 1.3 小组分工

| 小组 | 负责模块 | 依赖 | 技术要点 | 任务书 |
|------|----------|------|----------|--------|
| 老师演示 | 基础数据（科室/医生/排班） | 无 | 完整 CRUD 演示 | `DOC/tasks/task-teacher-基础数据演示.md` |
| A 组 | 患者管理 | 无 | TableView + Dialog 复用 | `DOC/tasks/task-A-患者管理.md` |
| B 组 | 挂号预约 | A 组 + 老师 | 事务控制、弹窗联动 | `DOC/tasks/task-B-挂号预约.md` |
| C 组 | 医生工作站 + 病历 | B 组 + 老师 | 状态机、表单 | `DOC/tasks/task-C-医生工作站与病历.md` |
| D 组 | 药品库存 + 处方 | C 组 | 操作列、复杂事务 | `DOC/tasks/task-D-药品库存与处方.md` |

### 1.4 模块依赖与接口约定

```
基础数据（老师） ─┬─→ B 组（挂号预约）─┬─→ C 组（医生工作站+病历）─┬─→ D 组（处方管理）
                │                     │                           │
A 组（患者）─────┘                     └───────────────────────────┘
```

各小组开发前必须**约定接口签名**（DAO 和 Service 的 public 方法），以便独立开发和后期集成。

## 二、Git 协作规范

### 2.1 仓库初始化（课时 1 完成）

```bash
# 老师创建中央仓库（GitHub / Gitee / GitLab）
git init --bare medicare.git

# 各小组克隆仓库
git clone <仓库地址>
```

### 2.2 分支策略

```
main （保护分支，仅通过 PR 合并）
  ├── feature/patient-mgmt      (A 组)
  ├── feature/registration      (B 组)
  ├── feature/doctor-workstation (C 组)
  └── feature/pharmacy          (D 组)
```

**规范**：
- `main` 分支受保护，禁止直接推送
- 每个模块一个 `feature/模块名` 分支
- 开发完成后发起 Pull Request，老师或其他组代码审查后合并

### 2.3 提交规范

```
feat: 新增患者管理模块 DAO 层
fix: 修复性别显示索引错误
refactor: 提取公共弹窗打开方法
docs: 更新接口文档
```

### 2.4 合并冲突处理

当两个小组同时修改了 `MainController.java`（添加导航方法注释）时：
1. 先 `git pull origin main` 获取最新代码
2. 如有冲突，手动编辑解决
3. `git add . && git commit` 提交解决后的代码

- **标准模板**：所有模块的数据加载都用这个模式

### 三、教学过程

#### 4.2 讲解：Task 标准模板（15 min）

**老师活动**：展示标准代码模板，强调"所有模块都用这个模板"：
```java
Task<List<Patient>> task = new Task<>() {
    @Override protected List<Patient> call() throws Exception {
        return patientService.listAll(); // 后台线程
    }
};
task.setOnSucceeded(e -> Platform.runLater(() -> {
    patientList.setAll(task.getValue()); // UI 线程更新
}));
task.setOnFailed(e -> Platform.runLater(() -> {
    showAlert("加载失败: " + task.getException().getMessage());
}));
new Thread(task).start();
```

#### 4.3 学生实操（25 min）

### 三、教学过程

#### 4.1 讲解：事务控制（20 min）

**老师活动**：以挂号为例，逐行走读：
```java
Connection conn = null;
try {
    conn = registrationDAO.getConnection();
    conn.setAutoCommit(false);
    scheduleDAO.decrementRemain(reg.getScheduleId());
    Long id = registrationDAO.insert(conn, reg);
    registrationDAO.commit(conn);
    return id;
} catch (SQLException e) {
    registrationDAO.rollback(conn);
    throw e;
} finally {
    registrationDAO.closeConnection(conn);
}
```

**强调**：B 组和 D 组的模块都涉及事务，必须掌握。

#### 4.2 学生练习（25 min）

**学生活动**：
- 以"取消挂号"为例，口述事务步骤
- 以"出入库"为例，手写事务代码框架

### 5.3 常见学员问题预案

|------|----------|
| "为什么不给完整代码？" | 脚手架已经提供了 60% 的框架代码，你们只需要写 DAO/Service/Controller/FXML，这正是需要学习的核心技能 |

| "Git 合并冲突了怎么办？" | 保留双方修改，冲突标记 `<<<<<` 要全部删除干净 |
| "我的模块依赖 A 组，A 组没做完" | 先用固定的测试数据开发，等 A 组完成后替换为真实接口调用 |
